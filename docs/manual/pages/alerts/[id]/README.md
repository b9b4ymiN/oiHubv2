# คู่มือหน้าแก้ไข Alert

- **Route:** `/alerts/[id]`
- **บทบาทของหน้า:** หน้าปรับแต่งและบำรุงรักษา alert rule

## เป้าหมายของหน้า
ใช้แก้ไข alert rule ที่มีอยู่แล้ว เช่น ปรับ threshold, เปลี่ยน channels, เปลี่ยน severity, หรือแก้ quiet hours หลังจากใช้งานจริงไปแล้ว

## ใช้เมื่อไร / เหมาะกับใคร
เปิดหน้านี้เมื่อ rule เดิมส่งสัญญาณมากเกินไป, น้อยเกินไป, หรือไม่สอดคล้องกับสภาพตลาดล่าสุด

## หน้านี้มีอะไรบ้าง
- ฟอร์มโครงสร้างเดียวกับหน้า create
- โหลดค่าปัจจุบันของ rule ตาม `id`
- มี state พิเศษสำหรับ `Loading` และ `Alert rule not found`

## ปุ่มและการโต้ตอบสำคัญ
- แก้ **Basic Information** และ **Conditions** ได้เหมือนหน้า create
- ปรับ **Channels / Throttle / Quiet Hours** ได้โดยไม่ต้องสร้าง rule ใหม่
- กด save เพื่ออัปเดตแล้วกลับไปหน้า list

## วิธีใช้งานที่แนะนำ
1. ใช้หลังจากมี evidence ว่า rule ปัจจุบัน noisy หรือช้าเกินไป
2. ปรับทีละตัวแปร เช่น threshold หรือ interval เพื่อให้ประเมินผลได้ชัด
3. หลังแก้แล้วกลับไป monitor ที่หน้า list ว่าเปิดใช้งานตรงตามที่ตั้งใจหรือไม่

## มุมมองแบบ OI Trader
หน้า edit คือจุดทำ **signal hygiene** ถ้า alerting stack ของคุณดีขึ้นเรื่อย ๆ คุณจะเสียเวลาหน้าจอน้อยลงและเหลือสมาธิไปกับ decision ที่สำคัญจริง

## ข้อควรระวัง
- อย่าปรับหลายอย่างพร้อมกันถ้าคุณยังไม่รู้ root cause ของ noise
- ถ้า rule ไม่พบ (`not found`) ให้กลับไปตรวจรายการใน `/alerts`
- การแก้ severity โดยไม่แก้ logic ไม่ได้เพิ่มคุณภาพของ signal เอง
